---
title: Vehicles and Routes
description: Vehicle inventory, lifecycle, odometer, engine-hours, counts, types, and route history endpoints.
---

Vehicle list endpoints support field projection through repeated `fields[]`
query parameters. Route endpoints can produce large responses when coordinates
are requested.

## GET /vehicles/fields/json

<ReferenceGrid>
  <div>
    <ReferenceHeader title="List vehicles" method="GET" path="/vehicles/fields/json" sdk="linxio.vehicles.list(params)">
      Returns a paginated list of vehicles.
    </ReferenceHeader>

    <FieldTable
      title="Query parameters"
      fields={[
        { name: "limit", type: "number", description: "Number of records per page." },
        { name: "page", type: "number", description: "Page number to read.", defaultValue: "1" },
        { name: "fields[]", type: "string[]", description: "Vehicle fields to include, such as id, regNo, status, lastLoggedAt, or lastCoordinates." },
        { name: "sort", type: "string", description: "Sort expression accepted by Linxio." },
      ]}
    />

    <FieldTable
      title="Response fields"
      fields={[
        { name: "data", type: "LinxioVehicle[]", description: "Vehicle rows for the current page." },
        { name: "page", type: "number", description: "Current page number." },
        { name: "limit", type: "number", description: "Page size." },
        { name: "total", type: "number", description: "Total matching vehicle count." },
        { name: "aggregations", type: "unknown", description: "Aggregation payload when supplied." },
      ]}
    />
  </div>

  <CodeRail>

```bash title="Request" lineNumbers
curl "https://api.linxio.com/api/vehicles/fields/json?fields[]=id&fields[]=regNo&limit=100&page=1" \
  -H "Authorization: Bearer $LINXIO_TOKEN"
```

```json title="Response" lineNumbers
{
  "page": 1,
  "limit": 100,
  "total": 2,
  "data": [
    { "id": 304, "regNo": "ABC-123" },
    { "id": 305, "regNo": "DEF-456" }
  ],
  "aggregations": null
}
```

  </CodeRail>
</ReferenceGrid>
