---
title: Reports and Forms
description: Scheduled report and digital form answer HTTP endpoints.
---

Report and digital form endpoints are dashboard-derived. Payload and response
fields can vary by enabled tenant features and form schemas.

    <FieldTable title="Response body fields" iconName="bodyParams" fields={[

## POST /scheduled-report

<ReferenceGrid>
  <div>
    <ReferenceHeader title="Create a scheduled report" method="POST" path="/scheduled-report" sdk="linxio.reports.createScheduled(payload)">
      Creates a scheduled report.
    </ReferenceHeader>

    <RequiredHeaders json />

    <FieldTable title="Request body parameters" iconName="urlParams" fields={[
      { name: "name", type: "string", required: true, description: "Report display name." },
      { name: "type", type: "string", required: true, description: "Report type accepted by Linxio." },
      { name: "format", type: "csv | pdf | xlsx", required: true, description: "Output format." },
      { name: "params", type: "Record<string, unknown>", description: "Report-specific parameters such as schedule, recipients, date range, filters, or vehicle IDs." },
    ]} />
    <FieldTable title="Response body fields" iconName="bodyParams" fields={[{ name: "id", type: "LinxioId", description: "Created scheduled report ID." }]} />
  </div>

  <CodeRail>

```bash title="Request" lineNumbers
curl -X POST https://api.linxio.com/api/scheduled-report \
  -H "Authorization: Bearer $LINXIO_TOKEN" \
  -H "Content-Type: application/json" \
  --data '{
    "name": "Weekly Fleet Summary",
    "type": "fleet-summary",
    "format": "pdf",
    "params": {
      "timezone": "Australia/Perth"
    }
  }'
```

  </CodeRail>
</ReferenceGrid>
